## Overview

This guide covers deploying a production Mempool instance similar to [mempool.space](https://mempool.space), supporting Bitcoin (mainnet, testnet, signet) and Liquid (mainnet, testnet). The mempool.space production site runs on FreeBSD with ZFS, but Linux works well too.

<Info>
  Professional support for production instances is available exclusively to project sponsors through [Mempool Enterprise®](https://mempool.space/enterprise). The Mempool team can also host a high-performance instance for you on their global infrastructure.
</Info>

## Hardware Requirements

Mempool v3 is powered by [mempool/electrs](https://github.com/mempool/electrs), which requires substantial resources.

### Recommended Specifications

## Operating System

The official mempool.space site runs on **FreeBSD 13** with ZFS for maximum performance and reliability.

<Tabs>
  <Tab title="FreeBSD 13 (Recommended)">
    ### Why FreeBSD?
    
    - Native ZFS support with superior performance
    - Advanced networking stack
    - Rock-solid stability
    - Efficient resource management
    
    ### Installation
    
    1. Download FreeBSD 13 from [freebsd.org](https://www.freebsd.org/)
    2. Install with ZFS root filesystem
    3. Configure ZFS for optimal performance (see filesystem section below)
  </Tab>
  
  <Tab title="Linux">
    ### Supported Distributions
    
    - Ubuntu 22.04 LTS or later
    - Debian 12 or later
    - CentOS Stream 9
    - Rocky Linux 9
    
    ### Recommended Filesystem
    
    - **ZFS on Linux** (OpenZFS) for advanced features
    - **ext4** or **XFS** for simpler setup
    - **Btrfs** is also supported
    
    <Note>
      While Linux works well, FreeBSD + ZFS provides the best performance for high-traffic production deployments.
    </Note>
  </Tab>
</Tabs>

## Filesystem Configuration

### ZFS Setup (FreeBSD)

For maximum performance, use **2x 2TB NVMe SSDs in RAID 0** with ZFS:

```bash ZFS Pool Creation
zpool create -f -o ashift=12 nvm mirror nvd0 nvd1
zpool list -v
```

Expected output:
```
NAME        SIZE  ALLOC   FREE  CKPOINT  EXPANDSZ   FRAG    CAP  DEDUP    HEALTH
nvm        3.62T  1.25T  2.38T        -         -     2%    34%  1.00x    ONLINE
  nvd0p3   1.81T   629G  1.20T        -         -     2%  33.9%      -  ONLINE
  nvd1p3   1.81T   646G  1.18T        -         -     2%  34.8%      -  ONLINE
```

### Partition Layout

Create separate ZFS datasets for flexibility and performance:

<CodeGroup>
```bash Bitcoin Core Datasets
zfs create nvm/bitcoin
zfs create nvm/bitcoin/blocks
zfs create nvm/bitcoin/chainstate
zfs create nvm/bitcoin/electrs
zfs create nvm/bitcoin/indexes
zfs create nvm/bitcoin/testnet3
zfs create nvm/bitcoin/testnet3/blocks
zfs create nvm/bitcoin/testnet3/chainstate
zfs create nvm/bitcoin/testnet3/indexes
```

```bash Electrs Datasets
zfs create nvm/electrs
zfs create nvm/electrs/mainnet
zfs create nvm/electrs/mainnet/cache
zfs create nvm/electrs/mainnet/history
zfs create nvm/electrs/mainnet/txstore
zfs create nvm/electrs/testnet
zfs create nvm/electrs/testnet/cache
zfs create nvm/electrs/testnet/history
zfs create nvm/electrs/testnet/txstore
zfs create nvm/electrs/signet
```

```bash Elements/Liquid Datasets
zfs create nvm/elements
zfs create nvm/elements/electrs
zfs create nvm/elements/liquidv1
zfs create nvm/electrs/liquid
zfs create nvm/electrs/liquid/cache
zfs create nvm/electrs/liquid/history
zfs create nvm/electrs/liquid/txstore
```

```bash Mempool & Database
zfs create nvm/mempool
zfs create nvm/mysql
```
</CodeGroup>

<Info>
  This partition structure allows independent snapshots, quotas, and compression settings per dataset.
</Info>

## Tor Configuration

For onion service support:

```bash
pkg install -y tor
pw user mod bitcoin -G _tor
```

### /usr/local/etc/tor/torrc

```ini torrc
RunAsDaemon 1
SOCKSPort 9050
ControlPort 9051
Log notice syslog

CookieAuthentication 1
CookieAuthFileGroupReadable 1
CookieAuthFile /var/db/tor/control_auth_cookie
DataDirectory /var/db/tor
DataDirectoryGroupReadable 1

# Mempool onion service
HiddenServiceDir /var/db/tor/mempool
HiddenServicePort 80 127.0.0.1:81
HiddenServiceVersion 3

# Liquid onion service
HiddenServiceDir /var/db/tor/liquid
HiddenServicePort 80 127.0.0.1:83
HiddenServiceVersion 3
```

Start Tor:
```bash
service tor start
```

Symlink Mempool nginx configs:

## Process Management

### systemd Services (Linux)

Create service files for each network:

```ini /etc/systemd/system/mempool-mainnet.service
[Unit]
Description=Mempool Backend (Mainnet)
After=network.target mariadb.service

[Service]
Type=simple
User=mempool
WorkingDirectory=/mempool/mainnet/backend
Environment="MEMPOOL_CONFIG_FILE=/mempool/mainnet/mempool-config.json"
ExecStart=/usr/bin/npm run start
Restart=on-failure
RestartSec=10
StandardOutput=append:/var/log/mempool/mainnet.log
StandardError=append:/var/log/mempool/mainnet-error.log

[Install]
WantedBy=multi-user.target
```

Enable all services:
```bash
systemctl enable mempool-mainnet
systemctl enable mempool-testnet
systemctl enable mempool-signet
systemctl enable mempool-liquid
systemctl enable mempool-liquidtestnet
systemctl start mempool-mainnet
```

## Security Hardening

<Steps>
  <Step title="Firewall">
    Only expose necessary ports:
    ```bash
    # Allow HTTP/HTTPS
    ufw allow 80/tcp
    ufw allow 443/tcp
    
    # Deny everything else
    ufw enable
    ```
  </Step>

  <Step title="Fail2ban">
    Protect against brute force:
    ```bash
    apt-get install fail2ban
    systemctl enable fail2ban
    ```
  </Step>

  <Step title="Secure Credentials">
    - Use strong passwords for all services
    - Store credentials in environment variables
    - Restrict file permissions: `chmod 600 mempool-config.json`
  </Step>

  <Step title="Updates">
    Keep system packages up to date:
    ```bash
    # FreeBSD
    freebsd-update fetch install
    pkg update && pkg upgrade
    
    # Ubuntu/Debian
    apt-get update && apt-get upgrade
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="High CPU usage">
    - Check electrs is fully synced
    - Verify database indexes are created
    - Monitor slow queries: `mysqldumpslow /var/log/mysql/slow.log`
    - Increase `MEMPOOL_POLL_RATE_MS` to reduce polling frequency
  </Accordion>

  <Accordion title="Memory exhaustion">
    - Increase system RAM
    - Tune database buffer pool size
    - Reduce ZFS ARC cache if needed
    - Enable swap (with low swappiness)
  </Accordion>

  <Accordion title="Disk I/O bottleneck">
    - Upgrade to faster NVMe drives
    - Use ZFS compression
    - Separate partitions for different services
    - Monitor with: `iostat -x 1`
  </Accordion>

  <Accordion title="Network issues">
    - Check nginx error logs
    - Verify rate limiting isn't too aggressive
    - Monitor connections: `netstat -an | grep ESTABLISHED | wc -l`
    - Increase max connections if needed
  </Accordion>
</AccordionGroup>
