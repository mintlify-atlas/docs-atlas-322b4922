## Overview

Transaction Acceleration helps you speed up stuck Bitcoin transactions by using CPFP (Child-Pays-For-Parent) to increase the effective fee rate, ensuring faster confirmation.

<img src="/images/acceleration-dashboard.png" alt="Transaction acceleration interface" />

## Why Transactions Get Stuck

Your Bitcoin transaction might be stuck because:

- **Fee too low**: Paid less than current market rate
- **Mempool congestion**: Network activity increased after sending
- **Transaction size**: Large transactions need higher total fees
- **No RBF**: Transaction not marked for Replace-By-Fee

<Info>
  "Stuck" means your transaction is in the mempool but not confirming. It's not lost - just waiting for miners to include it in a block.
</Info>

## How Acceleration Works

### Child-Pays-For-Parent (CPFP)

CPFP is the most reliable way to accelerate transactions:

<Steps>
  <Step title="Create Child Transaction">
    Spend an output from the unconfirmed (parent) transaction
  </Step>
  
  <Step title="Pay Higher Fee">
    Child transaction pays enough to cover both transactions
  </Step>
  
  <Step title="Package Mining">
    Miners include both transactions together to claim the combined fee
  </Step>
  
  <Step title="Confirmation">
    Both parent and child confirm in the same block
  </Step>
</Steps>

<img src="/images/cpfp-diagram.png" alt="CPFP acceleration diagram" />

### How Mempool Accelerator Works

Mempool's acceleration service:

1. **Analyzes your transaction**: Calculates effective fee rate including ancestors
2. **Estimates acceleration cost**: Shows cost to reach target fee rate
3. **Coordinates with mining pools**: Partners with major pools for priority
4. **Guarantees inclusion**: Transaction confirmed in next available block from partner pools

## Understanding Acceleration Cost

### Cost Breakdown

Acceleration cost consists of:

```
Acceleration Cost = Base Fee + vSize Fee + Mining Priority
```

<Tabs>
  <Tab title="Base Fee">
    **Service Fee**
    
    Flat fee to process the acceleration:
    - Covers operational costs
    - Typically a few thousand sats
    - Same regardless of transaction size
  </Tab>
  
  <Tab title="vSize Fee">
    **Transaction Size Fee**
    
    Based on your transaction's size:
    - Larger transactions cost more
    - Includes ancestor transactions
    - Charged per virtual byte
  </Tab>
  
  <Tab title="Priority Fee">
    **Mining Pool Bid**
    
    The actual acceleration amount:
    - Makes your transaction competitive
    - Goes directly to miners
    - Based on current mempool conditions
    - Adjustable for urgency
  </Tab>
</Tabs>

### Example Calculation

```
Transaction Details:
- Original fee rate: 5 sat/vB
- Transaction size: 250 vB
- Original fee: 1,250 sats

Current Mempool:
- Next block fee rate: 25 sat/vB
- Target total fee: 6,250 sats
- Fee increase needed: 5,000 sats

Acceleration Cost:
- Base fee: 1,500 sats
- vSize fee: 1,000 sats (4 sat/vB)
- Priority bid: 5,000 sats
- Total: 7,500 sats (~$3 at $40k/BTC)
```

<Info>
  Acceleration is most cost-effective when mempool fees are moderately high. During extreme congestion, costs can be significant.
</Info>

## Comparison with Alternatives

### RBF (Replace-By-Fee)

<CardGroup cols={2}>
  <Card title="RBF Advantages" icon="plus">
    - Cheaper (just pay fee difference)
    - Sender controls process
    - No third party needed
    - Works with any transaction
  </Card>
  
  <Card title="RBF Limitations" icon="minus">
    - Must be enabled when sending
    - Requires access to sending wallet
    - Not available after sending
    - Receiver can't initiate
  </Card>
</CardGroup>

<Info>
  **Best Practice**: Enable RBF on all transactions. Use acceleration only when RBF isn't available.
</Info>

### CPFP (Child-Pays-For-Parent)

<CardGroup cols={2}>
  <Card title="Self CPFP Advantages" icon="plus">

    - Works without RBF
    - Full control
  </Card>
  
  <Card title="Self CPFP Limitations" icon="minus">
    - Complex to execute
    - Need technical knowledge
    - Must calculate fees correctly
    - Requires compatible wallet

## Frequently Asked Questions

<AccordionGroup>
  <Accordion title="How long until my transaction confirms?">
    With ~40% hashrate coverage from partner pools:
    - **Average**: 20-30 minutes
    - **Best case**: Next block (10 min)
    - **Worst case**: If non-partner finds next few blocks (60+ min)
    
    Statistically, partner pools should mine a block within 25 minutes.
  </Accordion>
  
  <Accordion title="What if it doesn't confirm?">
    Acceleration continues until confirmation:
    - Your transaction stays prioritized
    - No additional payment needed
    - Guaranteed inclusion in partner pool blocks
    - Confirmation may just take longer if non-partners mine blocks
  </Accordion>
  
  <Accordion title="Can I accelerate someone else's transaction?">
    Yes, if:
    - You're the recipient (can do CPFP)
    - You have an output you can spend
    - You're willing to pay the cost
    
    Common for exchanges/merchants helping customers.
  </Accordion>
  
  <Accordion title="What happens if someone else accelerates my transaction?">
    Good news:
    - Your transaction benefits
    - No cost to you
    - Will confirm faster
    - Nothing you need to do
  </Accordion>
  
  <Accordion title="Can I cancel an acceleration?">
    No, once paid:
    - Acceleration is final
    - Transaction will be prioritized
    - Refunds not available
    - Transaction will eventually confirm
  </Accordion>
  
  <Accordion title="Is acceleration cheaper than using high fees initially?">
    Usually no:
    - Best to use correct fees from the start
    - Acceleration adds service fees
    - Use as last resort

    - Emergency RBF with very high fees

    - Customer support burden

## Technical Details

### Transaction Packages

CPFP works with transaction packages:

```
Package:
- Parent: Your stuck transaction
- Child: New transaction spending parent output
- Combined: Treated as single unit by miners

Effective Fee Rate:
= (Parent Fee + Child Fee) / (Parent Size + Child Size)
```

### Mining Pool Integration

How partner pools prioritize accelerations:

1. **Block Template**: Pools create block candidates
2. **Acceleration Injection**: Accelerated txs added with high priority
3. **Fee Optimization**: Still maximizes total block fees
4. **Broadcasting**: Block mined and broadcast to network

### Mempool Policy

Acceleration respects Bitcoin consensus rules:

- No double-spends
- Valid signatures
- Proper fee payment
- Standard transaction types

<Note>
  Acceleration can't help with invalid transactions or double-spend conflicts.
</Note>
