## Fastest Path: One-Click Installation

The quickest way to get Mempool running is through a one-click installation on a Bitcoin full-node distribution. This method handles all configuration automatically.

<Steps>
  <Step title="Choose your platform">
    Mempool is available on these popular Bitcoin node distributions:
    
    - **Umbrel** - Most user-friendly, app store installation
    - **RaspiBlitz** - Terminal-based, great for Raspberry Pi
    - **StartOS** - Privacy-focused with service dependencies
    - **myNode** - Premium features with community edition
    - **RoninDojo** - Samourai Wallet integration
    - **nix-bitcoin** - Declarative NixOS configuration
  </Step>

  <Step title="Install from your platform's app store">
    Open your node's app store or package manager and search for "Mempool". Click install and wait for the installation to complete (usually 5-10 minutes).
  </Step>

  <Step title="Access your Mempool instance">
    Once installed, access Mempool through your platform's dashboard or navigate directly to:
    
    ```
    http://your-node-ip:4080
    ```
    
    <Note>The exact port may vary depending on your platform. Check your platform's documentation for the specific URL.</Note>
  </Step>
</Steps>

## Quick Start with Docker

If you have Docker installed and a Bitcoin Core node running, you can get Mempool up in minutes.

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/mempool/mempool.git
    cd mempool/docker
    ```
  </Step>

  <Step title="Configure Bitcoin Core connection">
    Edit `docker-compose.yml` to set your Bitcoin Core RPC credentials:
    
    ```yaml docker-compose.yml
    api:
      environment:
        CORE_RPC_HOST: "172.27.0.1"
        CORE_RPC_PORT: "8332"
        CORE_RPC_USERNAME: "mempool"
        CORE_RPC_PASSWORD: "mempool"
    ```
    
    <Warning>Replace `172.27.0.1` with your Bitcoin Core IP if it's not running on the Docker host.</Warning>
  </Step>

  <Step title="Start the containers">
    ```bash
    docker-compose up -d
    ```
    
    This will start three containers:
    - **web** - Nginx frontend server
    - **api** - Node.js backend
    - **db** - MariaDB database
  </Step>

  <Step title="Access Mempool">
    Open your browser and navigate to:
    
    ```
    http://localhost
    ```
    
    You should see the Mempool interface. It will start populating with data as transactions are detected.
  </Step>
</Steps>

## Prerequisites

Before installing Mempool, ensure you have:

<CardGroup cols={2}>
  <Card title="Bitcoin Core" icon="bitcoin">
    Fully synced Bitcoin Core node with `txindex=1` enabled
  </Card>
  <Card title="Hardware" icon="server">
    Minimum 4GB RAM, 50GB free disk space (more for long-term operation)
  </Card>
  <Card title="Electrum Server (Optional)" icon="bolt">
    For address lookup functionality - Fulcrum or Electrs
  </Card>
  <Card title="MariaDB (Auto-installed)" icon="database">
    Required for statistics and mining data - included in Docker setup
  </Card>
</CardGroup>

<Info>
  Address lookups require an Electrum Server. Without one, Mempool will function but won't support address queries. See the [Electrum Server configuration](/configuration/electrum-server) guide for setup instructions.
</Info>

## Verify Installation

Once Mempool is running, verify everything is working correctly:

<AccordionGroup>
  <Accordion title="Check the backend API">
    Test the backend API by accessing:
    
    ```bash
    curl http://localhost/api/v1/backend-info
    ```
    
    You should receive a JSON response with version and network information.
  </Accordion>

  <Accordion title="Monitor the logs">
    For Docker installations:
    
    ```bash
    docker-compose logs -f api
    ```
    
    You should see messages like:
    ```
    Mempool updated in 0.189 seconds
    Updating mempool
    ```
  </Accordion>

  <Accordion title="Check the graphs">
    Navigate to your Mempool instance and verify that:
    - The mempool graph is displaying transactions
    - Recent blocks are showing in the blocks list
    - Fee recommendations are displayed
    
    <Note>It may take a few minutes for the graphs to populate with data.</Note>
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Mempool not syncing">
    **Check Bitcoin Core connection:**
    ```bash
    docker-compose logs api | grep "bitcoin"
    ```
    
    Ensure Bitcoin Core RPC credentials are correct and `txindex=1` is enabled in `bitcoin.conf`.
  </Accordion>

  <Accordion title="Address lookups not working">
    Address lookups require an Electrum Server. Configure your Electrum Server connection in the backend config:
    
    ```yaml
    MEMPOOL_BACKEND: "electrum"
    ELECTRUM_HOST: "your-electrum-ip"
    ELECTRUM_PORT: "50002"
    ```
  </Accordion>

  <Accordion title="Database connection errors">
    Wait for the database container to fully initialize (about 30 seconds). Check database logs:
    
    ```bash
    docker-compose logs db
    ```
    
    If issues persist, try recreating the database volume:
    ```bash
    docker-compose down -v
    docker-compose up -d
    ```
  </Accordion>

  <Accordion title="High memory usage">
    Mempool's memory usage increases with mempool size. For production deployments, we recommend at least 8GB RAM. You can also enable clustering:
    
    ```json
    "SPAWN_CLUSTER_PROCS": 4
    ```
  </Accordion>
</AccordionGroup>
